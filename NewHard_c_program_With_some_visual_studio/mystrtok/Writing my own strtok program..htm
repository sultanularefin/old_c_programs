      char * mystrtok(char * str, const char * delim)
      {
          for(i=0; i<sizeof(str) && str[i]!='\0'; i++)
          {
              for(j=0; j<sizeof(delim) && delim[j]; j++)
              {        
                  if (str[i]==delim[j])

      #define strtok mystrtok;

      Remove the semicolon from the end of this line so that is looks like this:

      #define strtok mystrtok

      The error is due to the fact that after the preprocessor runs, your code has expanded to essentially this:

      strArray[0] = mystrtok;(line, " ");
      the type of "mystrtok" by itself is a pointer to the function.  Hence the error.

      On 9/18/2012 2:14 PM, Trdough wrote:

      char * mystrtok(char * str, const char * delim)
      {
           for(i=0; i<sizeof(str) && str[i]!='\0'; i++)

      sizeof(str) == 4, the size in bytes of the pointer. It's completely unrelated to the length of the string pointed to.

               for(j=0; j<sizeof(delim) && delim[j]; j++)

      Same here - sizeof(delim) is not the length of the string that delim points to.

      #define strtok mystrtok;

      The trailing semicolon here is the immediate source of the error, as Wyck points out.
